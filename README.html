Setting up the repo locally

  1. Clone this repo
  2. Create a src folder and clone BepInEx repository recursively pulling the submodules into it.
    For instance, you can use git clone --recurse-submodules https://github.com/BepInEx/BepInEx.git . inside the src folder.
  3. Run nuget restore in the src folder to restore packages.
  4. Install DocFX. Currently our CI uses DocFX 2.47, so you should consider using the same version. However, pick the version that works with your VS/MSBuild setup. It's suggested to download and install MSBuild 15 separately if you use Visual Studio 2019 or older (or use).
    NOTE Currently DocFX works best with MSBuild 15 (which ships with VS19) or mono.
  5. Run docfx --serve in this repository folder to build the documentation and automatically serve them on localhost:8080.
